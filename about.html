Optimal Routing:
AI vs Dijkstra

A demonstration of Multi-Objective Evolutionary Algorithms solving NP-Hard QoS-constrained network routing problems across a deeply complex multi-attribute ISP backbone.

The Algorithmic Divide

🧭 Dijkstra's Limit

Dijkstra mathematically guarantees the optimal path for a single metric. In an unconstrained pure cost search, it holds absolute supremacy.

> But routing is never single-metric.

> If an enterprise SLA requires packet delivery < 50ms, the absolute cheapest path (a 150ms satellite shortcut) is invalid.

> Dijkstra's greedy expansion cannot look back to optimize secondary constraints once it makes a path choice.

🧬 The Genetic Breakthrough

Genetic Algorithms don't follow rigid steps; they simulate evolution. By managing a population of paths, they can efficiently tunnel through NP-Hard constraints.

> Operators deployed:
- Stochastic Depth-Capped Random Walks
- Shared-Node Crossover & Bridge Repair
- Adaptive Subpath Mutation

> Capable of finding the minimum expected cost that safely exists within a constrained 3D manifold.

The Engine Upgrade: NSGA-II

The initial version of this project used a rudimentary weighted-sum fitness scalarization. We completely overhauled the core processing engine to use the Non-dominated Sorting Genetic Algorithm II to ensure highly concrete, mathematically sound multi-objective results.

1. Pareto Fronts

Instead of mixing Latency, Cost, and Reliability into an arbitrary single score, the engine computes Pareto dominance mapping all routes across true 3D objectives concurrently.

2. Crowding Distance

To avoid convergence around local optima, the algorithm measures structural density along the Pareto fronts. It forces offspring generation toward vast, unexplored regions of the network to ensure true global minimum discovery.

3. Constrained Domination

Violating a maximum latency no longer deletes the path instantly. NSGA-II grades the margin of violation. This allows paths to safely traverse temporarily 'invalid' regions to uncover optimal subpaths on the other side.

// Implementation of Constrained Domination
def _dominates(a, b):
    # Feasible ALWAYS beats infeasible
    if a.cv == 0 and b.cv > 0: return True
    if a.cv > 0 and b.cv == 0: return False
    
    # If both violate constraints, the lesser violation wins
    if a.cv > 0 and b.cv > 0: return a.cv < b.cv
    
    # Standard Pareto Check
    return any(a.obj[i] < b.obj[i] for i in range(3))
